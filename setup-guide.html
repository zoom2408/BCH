1
Create a Notion Integration
Get your secret token from Notion
🌐 Browser ▶

What you'll do

💡
A Notion Integration is like a "robot account" that can read your databases. You'll get a secret token that the sync script uses to authenticate.

Instructions

1
Open notion.so/my-integrations in your browser
2
Click "+ New integration"
3
Name it BCH Tracker → click Save
4
Copy the "Internal Integration Secret" — it starts with secret_...
5
Paste it somewhere temporary (Notes app is fine) — you'll use it in Step 3
⚠️
Keep it private — treat this token like a password. Never paste it in a public place or commit it to GitHub.
2
Create a Parent Page in Notion
Where your 3 databases will live
📝 Notion ▶

Instructions

1
In Notion, click "+ New page" in the left sidebar
2
Name it BCH Workspace (or any name you like)
3
Connect the integration: Click the ··· menu (three dots, top-right corner of the page) → scroll to Connections → find BCH Tracker → click it
🚫
Common mistake — this is NOT the Share button
"Share → Invite" is for adding people to a page.
Integrations use a different menu: ··· → Connections.
If you skip this, Step 3 will fail with a 400 error.

Where to find the Connections menu

··· menu (top-right of page)
✏️ Rename
🔗 Copy link
⭐ Add to Favorites
📤 Export
🔌 Connections ← click here
🗑️ Delete
4
Copy the page URL from your browser address bar — you'll paste it in Step 3

4
Add GitHub Secrets
So the sync workflow can access Notion
🐙 GitHub ▶
💡
You need a GitHub account and an empty repo created first. If you don't have one yet, go to github.com/new and create a repo named bch-tracker.

Where to add secrets

1
Open your GitHub repo in the browser
2
Click Settings tab (top of repo page)
3
In the left sidebar → Secrets and variables → Actions
4
Click "New repository secret" — add all 4 secrets from the Terminal output, one at a time

The 4 secrets to add

Secret name (exact)Value (from Terminal)
NOTION_TOKENstarts with secret_...
TASKS_DB_IDUUID printed after "Tasks created"
CONTENT_DB_IDUUID printed after "Content Calendar created"
ROADMAP_DB_IDUUID printed after "Product Roadmap created"
⚠️
Secret names are case-sensitive. Type them exactly as shown — NOTION_TOKEN not notion_token.
5
Push the Tracker to GitHub
Upload all files to your repo
💻 Terminal ▶
💡
Run these commands one at a time in the same Terminal window, still inside the tracker/ folder.

Initialise git and commit

git init
git add .
git commit -m "init: BCH tracker"

Connect to GitHub and push

1
Go to your GitHub repo in the browser → copy the commands shown under "…or push an existing repository"
2
They look like this — replace YOUR_USERNAME and YOUR_REPO:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main
💡
If prompted for a GitHub password, use a Personal Access Token, not your account password. Create one at github.com/settings/tokens with repo scope.
6
Enable GitHub Pages
Makes your tracker accessible at a public URL
🐙 GitHub ▶
1
Open your GitHub repo → click the Settings tab
2
Scroll down the left sidebar → click Pages
3
Under Source → select "Deploy from a branch"
4
Branch: main  ·  Folder: / (root) → click Save
5
Wait ~1 minute → refresh the page — your URL appears at the top:
https://YOUR_USERNAME.github.io/YOUR_REPO
7
Run the First Sync
Pull Notion data into your tracker for the first time
🐙 GitHub ▶
1
In your GitHub repo, click the Actions tab
2
In the left list, click "Sync Notion → JSON"
3
Click the "Run workflow" dropdown → click the green "Run workflow" button
4
Wait ~30 seconds → a green checkmark ✓ means it worked
✅
After this, syncing is automatic — the workflow runs every 30 minutes on its own. You can also trigger it manually any time.
⚠️
If the workflow fails (red ✗): click into the run → expand the failing step — the most common cause is a missing or misspelled GitHub Secret from Step 4.

cd "/Users/zoom09/Documents/Claude/Projects/BCH/tracker"

Open the tracker

4

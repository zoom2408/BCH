Create a Notion Integration
Get your secret token from Notion
🌐 Browser ▶

What you'll do

💡
A Notion Integration is like a "robot account" that can read your databases. You'll get a secret token that the sync script uses to authenticate.

Instructions

1
Open notion.so/my-integrations in your browser
2
Click "+ New integration"
3
Name it BCH Tracker → click Save
4
Copy the "Internal Integration Secret" — it starts with secret_...
5
Paste it somewhere temporary (Notes app is fine) — you'll use it in Step 3
⚠️
Keep it private — treat this token like a password. Never paste it in a public place or commit it to GitHub.

📝 Notion ▶

Instructions

1
In Notion, click "+ New page" in the left sidebar
2
Name it BCH Workspace (or any name you like)

🔧 Troubleshooting
!
Tracker shows 0 items / dashboard is empty
Sync ran (green ✓) but data/x.json has "total": 0, "items": []
▶
⚠️
Root cause: The DB ID saved in GitHub Secrets is pointing to the wrong database (or the parent page instead of the database itself). The API connects fine but returns 0 rows.

Step 1 — Get the correct DB ID from Notion

1
Open the database in Notion — click into the Tasks database itself (not just the parent page)
2
Look at the browser URL bar. It will look like one of these:
# Format 1 — long URL with page name:
https://www.notion.so/Tasks-b36f625eaddd1815695b8e3fd8fb9f23
                              ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                              ← this 32-char string is your DB ID

# Format 2 — just the ID:
https://www.notion.so/b36f625e-addd-1815-695b-8e3fd8fb9f23
3
Copy the 32 characters after the last - before any ?. Dashes are optional — both formats work.
🚫
Don't copy the parent page URL. The parent page (e.g. "BCH Workspace") has its own ID — it's NOT the database ID. Make sure the URL says the database name (Tasks, Content Calendar, Roadmap) in the browser tab before copying.

Step 2 — Verify with a quick Terminal test

💡
Run this to confirm the ID is correct before updating GitHub. Replace the values with yours.
NOTION_TOKEN="secret_xxx" TASKS_DB_ID="your-db-id" python3 -c "
import urllib.request, json, os
t = os.environ['NOTION_TOKEN']
d = os.environ['TASKS_DB_ID']
req = urllib.request.Request(
    f'https://api.notion.com/v1/databases/{d}/query',
    data=b'{}',
    headers={'Authorization': f'Bearer {t}',
             'Notion-Version': '2022-06-28',
             'Content-Type': 'application/json'},
    method='POST'
)
with urllib.request.urlopen(req) as r:
    data = json.loads(r.read())
    print(f'Found {len(data[\"results\"])} rows')
"
Expected output
Found 3 rows  ← ✓ correct ID
Found 0 rows  ← ✗ wrong ID or integration not connected

2
Click the secret name (e.g. TASKS_DB_ID) → click "Update" → paste the correct ID → save
3
